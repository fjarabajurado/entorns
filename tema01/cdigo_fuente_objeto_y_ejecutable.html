Código Fuente, Objeto y Ejecutable

Código Fuente

El código fuente de un programa informático es un conjunto de líneas de texto con los pasos que debe seguir la computadora para ejecutar dicho programa.

El código fuente de un programa está escrito en algún lenguaje de programación (Basic, Java, PHP...) pero en este primer estado no es directamente ejecutable por el ordenador, sino que debe ser traducido a otro lenguaje o código binario; así será más fácil para la máquina interpretarlo (lenguaje máquina o código objeto que sí pueda ser ejecutado por el hardware de la computadora). Para esta traducción se usan los llamados compiladores, ensambladores, intérpretes y otros sistemas de traducción.

Un ejemplo de código fuente en Java es el siguiente:

/**
* @author Fran Jaraba
*/
public class LoginControl extends HttpServlet {
    @override
    protected void service (HttpServletRequest request, HttpServletResponse response)  throws ServletException, IOException {
        switch (request.getServletPath()) {
            case "/LogonIngresar":
                ingresar(request, response);
                break;
            case "/LogonSalir":
                salir(request, response);
                break;
        }
    }
}

Código Objeto

En programación, se llama código objeto al código que resulta de la compilación del código fuente.​ Puede ser en lenguaje máquina o bytecode, y puede distribuirse en varios archivos que corresponden a cada código fuente compilado. Luego un enlazador (linker) se encarga de juntar todos los archivos de código objeto para obtener el programa ejecutable.

  • Código máquina: puede ser ejecutado directamente por el procesador. Las instrucciones depende de las diferentes plataformas existentes, por lo que al compilar un programa se debe utilizar un compilador que genere código para la plataforma deseada.
  • Bytecode: requiere ser procesado antes de ser ejecutado por el ordenador.

El código objeto es más eficiente que el código fuente, ya que el proceso de compilación reduce el tamaño del código, haciéndolo más pequeño y directo. Además, los programas de código objeto son más seguros que los de código fuente, puesto que ya están traducidos y son mucho más difíciles de leer y descodificar.

Código Ejecutable

El código ejecutable es una traducción completa a código máquina, realizada por el enlazador (linker)  y que ya es directamente ejecutable.

Linker (montador o enlazador): es el programa encargado de insertar al programa objeto el código máquina de las funciones de las librerías (archivos de biblioteca) usadas en el programa y realizar el proceso de montaje, que producirá un programa ejecutable. Las librerias son una colección de código (funciones) ya programado y traducido a código máquina, listo para utilizar en un programa y que facilita la labor del programador.

La diferencia entre el código objeto y el código ejecutable es que en el código objeto solo está en código máquina (o bycode) el programa que el programador ha escrito, pero no las posibles librerías que haya utilizado. Imagina que un programa envia un documento a imprimir. En el código objeto estará la llamada a imprimir, pero no estará el código de los pasos a realizar para comunicarse con la impresora. Esos pasos estarán el el driver (librería de impresora) de la impresora que está en el SO. El Linker, junta todas las llamadas a funciones de librería para crear el archivo ejecutable.
